# X Thread 风格指南

目标平台：**X (Twitter) @FrankFu2262**
受众：全球开发者、加密行业从业者、科技关注者

## 核心原则

### 1. 第一条决定一切

90% 的人只看第一条推文。第一条的任务：

- **有 Hook** —— 反常识、悬念、数据冲击
- **不能是总结** —— 不要一上来就"今天我来讲 XXX"
- **280 字内**，最好 180 字内（留空给短链接）

**好的第一条示例：**

> Base、BSC、以太坊都在做 Meme，但 Solana 占了 90%+ 的市场。
>
> 作为在交易所做了 2 年 meme 交易基建的人，告诉你这不是"便宜+快"那么简单。
>
> 是三个优势叠加，让"Meme 交易"这件事从不可能变成了可规模化产业。🧵

**坏的第一条示例：**

> 今天我来聊聊 Solana 为什么赢了 meme 交易市场。

### 2. 每条独立有信息量

Thread 里每一条推文：

- 都应该独立有信息（读者只看这一条也能 get 到一个观点）
- 不要用"承上启下"的水推
- 280 字内（中文按 X 的规则，中文算 2 字符）

### 3. 长度建议

- 简短 Thread：3-5 条
- 标准 Thread：6-10 条
- 深度 Thread：10-15 条（很少超过）
- 每个 Thread **必须**有明确的"最后一条 CTA"

### 4. 视觉节奏

X 的排版工具有限，但要用好这些：

- `---` 分隔线（让阅读呼吸）
- Bullet points (`- ` / `• `)
- 适度 emoji（每条 1-2 个，不要泛滥）
- 加粗用首尾空行强调（X 不支持 `**bold**`）

## 改写规则

### 从博客改到 X Thread

1. **砍掉 50-70% 的内容**：博客的详细论证、例子、代码，大部分要砍
2. **保留最有冲击力的观点和数据**：每个核心观点一条推文
3. **把长段落变成 bullet 列表**：X 上列表比散文好读 10 倍
4. **数据单独成条**：一条推文只放一个数据/对比，视觉冲击力强
5. **加强开头**：博客的开场通常比 Thread 的开场温和太多

### 第一条模板

```
[反常识的事实或数据]

[为什么你应该听我说 - 身份证明]

[文章的核心论点浓缩版]

🧵
```

### 最后一条模板

```
完整文章：
blog.frankfu.cloud/posts/[slug]

《[系列名]》系列第 N 篇。@FrankFu2262
```

## 发布策略

- **最佳时间（北京时间）**：
  - 中午 12:00-14:00（午休黄金时段）
  - 晚上 20:00-22:00（下班后）
- **最佳时间（UTC）**：
  - 04:00 UTC（中午北京，下午欧洲）
  - 12:00 UTC（晚上北京，上午美东）
- 避开周末（阅读量低 50%+）
- 技术内容周二/周四表现最好

## 字符数注意事项

X 的中文字符规则（2026）：
- 中文：算 2
- 英文字母 / 数字：算 1
- Emoji：算 2
- 总上限：280

我们的 Thread 脚本里的校验就是按这个规则。
**写的时候每条 < 260 字比较安全**（留 20 字的容错）。

## 不要做的事

- ❌ 不要在一条推文里塞多个观点
- ❌ 不要用"今天给大家分享一下" 这种开场
- ❌ 不要长篇引用（超过 2 行的引用效果差）
- ❌ 不要 hashtag 堆砌（最多 1-2 个有意义的）
- ❌ 不要只发链接不加 hook（几乎没人点）
- ❌ 不要用 `**bold**`、`*italic*`（X 不渲染）
