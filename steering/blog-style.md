# 博客风格指南

目标平台：**blog.frankfu.cloud**
受众：开发者、工程师、Web3 / AI 从业者

## 风格要求

### 结构

- 有明确的 H2/H3 层次
- 开场（why）：3 段以内，说明"你凭什么看下去"
- 主体：分 3-5 个核心点，每点用小标题
- 结尾：给出明确的 takeaway 或下一步

### 语气

- **第一人称视角**：用"我"，不用"我们"
- **叙事 + 技术分析结合**：不要纯技术手册
- **专业但有温度**：承认复杂性，不装懂
- **有观点**：不做骑墙党

### 技术处理

- 代码片段要能直接运行，不用伪代码
- 数据要有来源（链接或时间戳）
- 架构用 ASCII 图或 Mermaid
- 专业术语首次出现给简短解释

### 长度

- 核心文章：1500-3000 中文字
- 深度长文：3000-5000
- 不要超过 5000，超过就拆系列

## 保留原文的部分

- 所有代码块
- 所有数据和引用
- Mermaid 图表
- 个人经历和观点

## 改写的部分

只在这些情况下改写：
- 结构调整（小标题层级、顺序）
- 语气统一（太随意或太正式都要调）
- 删除冗余（重复的观点合并）
- 加强钩子（开头和每个 H2 的第一句）

## 引流 CTA（结尾固定）

```
---

_[订阅 RSS](/rss.xml) 或 [关注 X](https://x.com/FrankFu2262)，不错过更新。_
```

## 示例（好的开场）

❌ "今天我来分享一下 XXX 技术..."

✅ "2024 年初，Pump.fun 上线。当时没人认真对待。两年后，它成了 Solana 上第一个累计收入 $10 亿的应用。

作为在里面做了两年平台方的人，我想把这个过程复盘一遍。"
