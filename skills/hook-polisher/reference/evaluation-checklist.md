# Hook 评估清单

改不改一个 hook，看这个清单。

---

## 3 秒测试（终极标准）

模拟读者在手机上滑动时看到这个 hook，问自己：

> **"我会停下来读下面的内容吗？"**

- **"会"** → 合格
- **"可能会"** → 改
- **"不会"** → 必改

---

## 9 条具体检查

每一条回答 yes/no：

### 1. 有具体数字或时间吗？

- "$10B" / "2024 年 3 月" / "90%" → yes
- "很多" / "最近" / "一些" → no

### 2. 有具体身份吗？

- "交易所后端 2 年" / "前 Google 工程师" → yes
- "我" / 没说 → no

### 3. 有反差或冲突吗？

- "A 便宜，但 B 赢了" / "表面 X，实际 Y" → yes
- 平铺陈述 → no

### 4. 字数在上限内吗？

每个平台的上限见 `hook-rules-per-platform.md`。

### 5. 读得顺吗？

口语化、有节奏。
- 念出来卡壳 → no

### 6. 首字有冲击力吗？

- "$10B" / "凌晨 3 点" / "Solana 吃掉" → yes
- "今天" / "最近" / "作为" / "关于" → no

### 7. 有作者的声音吗？

- 能从 hook 里感受到"这是 Frank 写的" → yes
- 像任何 AI 都能写出来 → no

### 8. 不过度吗？

- "震撼！" / "颠覆！" / "史上最！" / "不看后悔" → no
- 专业但直白 → yes

### 9. 和 takeaway 一致吗？

hook 承诺的方向必须和文章内容的 takeaway 一致。标题党 = 失败。

---

## 评分

- 9/9：直接 pass
- 7-8/9：小改
- 4-6/9：重写
- < 4/9：hook 完全失败，回到 outline 找 hook 候选

---

## 常见失败模式

### 模式 1：像广告文案

```
❌ "Solana，颠覆传统金融的王者，不容错过！"
✅ "Solana 吃掉 meme 交易 90%+ 市场。不是'便宜+快'那么简单。"
```

### 模式 2：信息量太低

```
❌ "Web3 后端很有挑战性。"
✅ "Web3 后端每 3 个月一次技术范式切换，学习成本是 Web2 的 3 倍。"
```

### 模式 3：像摘要

```
❌ "本文将讨论 Web3 和 Web2 的 5 个差异。"
✅ "在大厂 4 年，转 Web3 1 年，多赚了 10 年工资。但别急着转。"
```

### 模式 4：AI 味过重

```
❌ "在当今这个瞬息万变的科技时代，Web3 作为新兴的范式..."
✅ "2024 年 1 月 Pump.fun 上线。18 个月后它赚了 $10B。"
```

### 模式 5：过长

```
❌ [250 字的长 hook]
✅ [50 字的断言 hook]
```

**原则**：写完后**尝试再砍 30%**。砍不掉了再止。
