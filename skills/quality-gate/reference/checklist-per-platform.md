# 各平台 Checklist

---

## Blog (`generated/<slug>/blog.md`)

### Frontmatter

- [ ] `title` 有引号 + 不超 40 字
- [ ] `slug` kebab-case，和目录一致
- [ ] `pubDatetime` ISO 8601 带时区（UTC）
- [ ] `tags` 2-4 个，在 `blog/TAGS.md` 已有的范围内
- [ ] `description` 140 字内、能独立成句
- [ ] `author` = "Frank Fu"
- [ ] `draft` 默认 false（发布前确认）

### 内容

- [ ] Hook 段落在 3 段以内
- [ ] 有 3-5 个 H2（论点数）
- [ ] 每个 H2 section 200-600 字
- [ ] 所有数据带来源（链接 / 时间戳 / 引用）
- [ ] 代码块可运行（或明确标注 `// 示意`）
- [ ] 术语首次出现有括号解释
- [ ] 段落短（超过 5 行的必砍）
- [ ] 总字数 1500-3000（深度文 3000-5000）
- [ ] 结尾有明确 takeaway
- [ ] 结尾有 CTA（订阅 RSS 或关注 X）

### 声音

- [ ] 第一人称 "我"（没有"我们"）
- [ ] 没有"今天我来聊聊"开场
- [ ] 没有"综上所述"结尾
- [ ] 没有明显 AI 味（"在当今"、"毫无疑问"）
- [ ] 没有形容词堆砌

### 合法性

- [ ] 没有未证实的八卦
- [ ] 没有攻击具体个人
- [ ] 没有歧视 / 政治站队
- [ ] 没有泄露雇主机密
- [ ] 没有硬广

---

## X Thread (`generated/<slug>/x-thread.md`)

### 结构

- [ ] 6-15 条之间
- [ ] 每条有 `## N/` 编号
- [ ] 最后一条有 `(CTA)` 标注

### 第一条

- [ ] < 180 字（中文 2 字符规则）
- [ ] 有反常识 / 数据 / 故事 / 身份钩子之一
- [ ] 结尾 🧵
- [ ] 不依赖后面推文也能独立成段

### 每条

- [ ] < 260 字
- [ ] 独立有信息量（不是过渡推）
- [ ] 没有 `**bold**` / `*italic*`
- [ ] bullet 用 `- ` 或 `• `，不用数字列表

### 最后一条 (CTA)

- [ ] 有博客链接（blog.frankfu.cloud）
- [ ] 有互动引导（具体问题，不是"你觉得呢"）
- [ ] 有 @FrankFu2262 自引用

### 合法性

- [ ] 没有硬广 / 带货
- [ ] 没有未证实的指控
- [ ] hashtag ≤ 2 个，且相关

## 小红书 (`generated/<slug>/xiaohongshu.md`)

### 标题

- [ ] 15-25 字
- [ ] 带数字、身份、反差、悬念、金钱中的至少 2 个
- [ ] 不是博客标题直接搬
- [ ] 不是"聊聊 XX" / "浅谈 XX"
- [ ] 不是标题党（"必看"、"爆款"）

### 开场

- [ ] 第一行有 emoji 开头
- [ ] 身份锚定或数据开场
- [ ] 短段落（1-3 句）
- [ ] 结尾有 `⬇️` 或引向下文

### 主体

- [ ] 3-5 个小标题
- [ ] 每个小标题 emoji 开头
- [ ] 每个段落 1-3 句
- [ ] 所有英文术语首次出现有括号中文解释
- [ ] 没有大段英文
- [ ] 没有大段代码
- [ ] 字数 500-800

### 结尾

- [ ] 有 CTA（关注主页 / 评论互动）
- [ ] 没有贴外链

### 标签

- [ ] 5-10 个
- [ ] 包含 2-3 个垂直 + 2-3 个身份 + 2-3 个泛
- [ ] 没有打错字
- [ ] 标签区写了两份（frontmatter + 正文末尾）

### 图文建议

- [ ] 有封面图建议
- [ ] 正文图 0-5 张，每张有描述

## 抖音 (`generated/<slug>/douyin-script.md`)

### Hook (`[0:00-0:03]`)

- [ ] 20-30 字
- [ ] 3 秒内能说完
- [ ] 首字有冲击（不是"今天"、"最近"）
- [ ] 有视觉提示

### 价值承诺 (`[0:03-0:15]`)

- [ ] 60-80 字
- [ ] 有身份锚定
- [ ] 明确预告几个要点

### 主体

- [ ] 3-5 个要点
- [ ] 每个要点 25-40 秒（60-100 字）
- [ ] 每句 10-15 字
- [ ] 口语化（没"然而"、"因此"、"综上"）
- [ ] 每段有视觉提示
- [ ] 所有技术术语有解释

### CTA

- [ ] 有"关注我"或明确行动指令
- [ ] 有下期预告（具体，不模糊）
- [ ] 没有塞外链

### 总时长

- [ ] 2-3 分钟（600-900 字）

### 视觉

- [ ] 每段有 `（视觉：xxx）` 标注
- [ ] Hook 处视觉有冲击力
- [ ] CTA 处有关注按钮视觉

---

## 跨平台一致性检查

- [ ] takeaway 在 4 个平台是同一个
- [ ] 核心论点在 4 个平台对齐（不能小红书讲 A，抖音讲 B）
- [ ] 作者声音统一（专业 + 有温度 + 不装）
- [ ] 数据在 4 个平台一致（"90%" 不能变成 "大部分"）
