# Thread 结构

选定一个结构型号再写。不要混着来。

---

## 浓缩型（6-8 条）

用于：短博客、单一观点、hot take

```
1/ Hook（180 字内）
2/ 论点 1 + 核心证据
3/ 论点 2 + 核心证据
4/ 论点 3 + 核心证据
5/ （可选）论点 4
6/ Takeaway（一句话总结）
7/ CTA（博客链接 + 互动）
```

---

## 标准型（9-12 条）

用于：中等深度博客、多数场景默认选这个

```
1/ Hook
2/ 铺垫（为什么这个话题现在重要）
3/ 论点 1：论点句
4/ 论点 1：证据 / 例子
5/ 论点 2：论点句
6/ 论点 2：证据 / 例子
7/ 论点 3：论点句
8/ 论点 3：证据 / 例子
9/ （可选）论点 4
10/ Takeaway
11/ CTA
```

---

## 深度型（13-15 条）

用于：超长博客、系列文首篇

```
1/ Hook
2/ 铺垫
3/ 论点 1：论点句
4/ 论点 1：证据 1
5/ 论点 1：证据 2
6/ 论点 2：论点句
7/ 论点 2：证据
8/ 论点 3：...
9-12/ 论点 4-5
13/ 全文回顾一句话
14/ Takeaway
15/ CTA
```

**警告**：超过 15 条的 Thread 完读率急剧下降。能压到 12 条就压。

---

## 每条的内部结构

### 标准条

```
[论点句或观察 — 1-2 句]

- [bullet 1]
- [bullet 2]
- [bullet 3]

[可选：一句数据 / 对比]
```

### 数据条

```
[一个数字 / 对比]

- A: xxx
- B: yyy
- 差距: zzz
```

### 故事条

```
[具体时间 + 一件事]

[一句话的结论]
```

### CTA 条

```
写了 [长度] 字完整分析：
blog.frankfu.cloud/posts/[slug]

[一句话互动]

@FrankFu2262
```

---

## 过渡

原则：**不要过渡推文**。

每条独立就意味着读者从中间任何一条点开都能 get 到信息。

如果真觉得论点 2 和论点 3 之间读者会卡住，在论点 3 的首句加一个"另一个角度是..."带过去，不单独成条。

---

## 节奏

- 前 3 条：**高密度**（吸引留下）
- 中间：**信息均匀**
- 倒数第 2-3 条：**高潮**（一个最扎实的数据 / 金句）
- 最后一条：**CTA**（不要忘了放链接）

---

## 决定用哪个结构

| 博客字数 | Thread 结构 |
|---|---|
| <1500 字 | 浓缩型 |
| 1500-3000 字 | 标准型 |
| 3000+ 字 | 深度型（但先考虑能不能拆系列 Thread） |

outline.md 里的"预计长度"能直接告诉你。
