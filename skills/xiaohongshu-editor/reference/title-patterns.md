# 小红书标题模板

标题是小红书的第一生产力。80% 的流量靠标题拉。

---

## 5 个高转化模板

### 1. 数字型

```
[数字] + [身份/主题] + [结果]
```

**示例：**
- "交易所 4 年，我转 Web3 第 1 年多赚了 30 万"
- "Web3 后端踩的 5 个大坑，第 3 个让我赔了 5 万"
- "3 分钟看懂 Solana 凭什么吃下 meme 90% 的市场"

### 2. 身份型

```
[我的身份] + [经验/结论]
```

**示例：**
- "交易所工程师告诉你，meme 币是怎么割韭菜的"
- "在 Web3 做后端 2 年，我终于敢说一句实话"
- "互联网后端转 Web3 一年，我的实战总结"

### 3. 反差型

```
[大家以为 X]，[其实 Y]
```

**示例：**
- "你以为 Web3 门槛高，其实比你想的更高"
- "大厂都在做 AI，但真正赚钱的是另一批人"
- "Solana 赢了 meme，但不是因为"快"，是因为..."

### 4. 悬念型

```
[具体场景] + [悬念]
```

**示例：**
- "Solana 上有个秘密赌场，90% 的人不知道..."
- "我的 Web3 面试遇到一个题，问倒了 5 年后端"
- "交易所凌晨 3 点那场事故，起因是一行代码"

### 5. 清单型

```
[N] 个 + [东西] + [价值感]
```

**示例：**
- "转 Web3 必看的 5 本书，都是工程师视角"
- "Web3 后端开发的 7 个小工具，第 3 个神器"
- "Solana RPC 优化的 4 个笨办法，我靠这个省了 $3k"

---

## 标题长度

**15-25 字最佳**。

- < 15 字：信息量不够，读者划走
- > 25 字：小红书列表页会截断

---

## 必备元素（至少 2 个）

- 数字（"3 个"、"30 万"、"第 5 天"）
- 具体身份（"交易所工程师"、"Web3 后端"、"大厂"）
- 反差 / 冲突（"其实"、"但是"、"没想到"）
- 悬念 / 好奇（"..."、"你不知道"、"颠覆"）
- 金钱 / 数字价值（"赚了 X 万"、"省了 X"）

---

## 禁止

- ❌ "聊聊 Web3"
- ❌ "浅谈 Solana 架构"
- ❌ 纯技术术语标题（"MEV 和 Jito 的关系详解"）
- ❌ 过度营销标题（"必看！"、"爆款！"、"史上最全！"）
- ❌ 封闭式问号标题（"你知道 Web3 吗？"）

---

## 怎么从博客标题转换

博客标题：**Solana 为什么赢了 meme 交易市场**

小红书标题（可选几个）：
1. "做 Web3 基建 2 年，我终于明白 Solana 为什么赢 meme"（身份型）
2. "Solana 赢 meme 的 3 个原因，'快'只排第 3"（数字 + 反差）
3. "Base、BSC 都在做，但 meme 市场 90% 在 Solana，凭什么？"（悬念）

从 3 个里挑一个最扎眼的。**不要博客标题直接搬**。
