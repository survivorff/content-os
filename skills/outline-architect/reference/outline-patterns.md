# 骨架模式

5 种高复用的内容结构。选一个，不要融合。

---

## 1. 问题解决型 (Problem-Solution)

**用在**：技术深度、故障复盘、架构文章

```
Hook：一个痛点场景（读者能共情）
Body:
  - 论点 1：问题的根源（不是表象）
  - 论点 2：常见但错误的解法 + 为什么错
  - 论点 3：正确解法 + 原理
  - 论点 4：上线后的效果 / 坑
Takeaway：一句话的经验总结
```

---

## 2. 故事驱动型 (Narrative)

**用在**：转型故事、一手经历、行业内幕

```
Hook：一个具体时间、地点、事件（给画面）
Body:
  - 论点 1：事件发生前的背景
  - 论点 2：关键转折点
  - 论点 3：结果和意外
  - 论点 4：从中抽出的可复用观察
Takeaway：超越这个故事本身的意义
```

---

## 3. 反常识型 (Contrarian)

**用在**：行业观察、观点输出、有"揭秘"感的话题

```
Hook：大家都以为 X，其实 Y
Body:
  - 论点 1：为什么大家会误解（表象 / 传播机制）
  - 论点 2：证据 1（硬数据）
  - 论点 3：证据 2（案例）
  - 论点 4：证据 3（逻辑推演）
Takeaway：这个 Y 对读者意味着什么
```

---

## 4. 对比型 (Compare)

**用在**：技术选型、职业决策、工具对比

```
Hook：用具体场景引出 A vs B 的选择
Body（每个论点是一个对比维度）:
  - 论点 1：维度 1 对比 + 谁赢
  - 论点 2：维度 2 对比 + 谁赢
  - 论点 3：维度 3 对比 + 谁赢
  - 论点 4：什么情况选哪个（明确建议）
Takeaway：给决策者的一句话
```

---

## 5. 清单型 (Listicle)

**用在**：经验总结、踩坑 N 条、工具集合

```
Hook：为什么现在要看这个清单
Body（每个论点是一条）:
  - 第 1 条：标题 + 1 句话 + 1 个例子
  - 第 2 条：标题 + 1 句话 + 1 个例子
  - 第 3 条：...
  - 第 N 条：...（总数 3-7 最佳）
Takeaway：把这些串起来的元规律
```

**清单型特别注意**：每条长度尽量均匀。如果有一条特别长，说明那条应该独立成篇。

---

## 如何选

| 信号 | 选 |
|---|---|
| 作者想讲"我们怎么解决 XX" | 问题解决型 |
| 作者想讲"某次我经历了 XX" | 故事驱动型 |
| 作者想讲"大家都错了，其实 XX" | 反常识型 |
| 作者想讲"A 还是 B" | 对比型 |
| 作者想讲"我总结了 N 条" | 清单型 |

不要勉强。想法里哪种气质最强就用哪个。

---

## 跨模式融合（慎用）

偶尔可以融合，例如：
- 反常识型 + 故事驱动型 = 以一个故事引出反常识
- 对比型 + 清单型 = 用一个维度清单做对比

但不要超过 2 种模式融合。融合太多就成散文了。
