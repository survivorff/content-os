# Outline 输出格式

写到 `contents/<slug>/outline.md`。

---

## 模板

```markdown
---
slug: kebab-case-slug
pattern: problem-solution | narrative | contrarian | compare | listicle
recommended_title: <从 scout 的候选里挑一个>
target_audience: <从 scout 抄过来>
takeaway: <从 scout 抄过来>
date_outlined: YYYY-MM-DD
---

# Outline

## 推荐标题

**<标题>**

（备选：见 scout.md）

## Hook 候选（3 选 1 后续由 hook-polisher 打磨）

### 选项 A — 数据钩
<50-100 字>

### 选项 B — 故事钩
<50-100 字>

### 选项 C — 反差钩
<50-100 字>

推荐：**A/B/C**，理由：<一句话>

## 骨架

### 论点 1：<论点句>

**支撑：**
- <具体数据 / 代码 / 经历>
- <可选第二条>

<可选：过渡>

### 论点 2：<论点句>

**支撑：**
- <具体>

### 论点 3：<论点句>

**支撑：**
- <具体>

### (论点 4/5 按需)

## 结尾

**Takeaway**：<一句话>

**CTA**：<订阅 / 关注 / 下期预告 / 评论互动>

## 风险提示

- <比如"论点 2 的数据需要作者再确认来源">
- <比如"论点 3 涉及的经历要脱敏">

## 预计长度

- 博客：<1500-3000 字>
- X Thread：<6-10 条>
- 小红书：<500-800 字>
- 抖音：<2-3 分钟 / 600 字脚本>
```

---

## 注意

- **推荐标题只推荐一个**，不要给作者做选择题。
- **Hook 候选一定要 3 个**，不要偷懒。
- **论点数量严格 3-5**。
- **风险提示一定要填**，没风险就写"暂无"。
- **预计长度**是给后面 skill 用的，不要漏。
