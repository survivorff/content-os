不同想法适合不同平台。用这个矩阵初筛。

## 单平台特征

### Blog
- **吸收**：任何长内容
- **回报**：SEO、权威感、作者 IP 沉淀
- **频率**：1-2 周一篇
- **每篇必发**：其他平台可选，blog 是默认 on

### X (Twitter)
- **吸收**：技术观察、行业 hot take、长文缩写
- **回报**：涨粉快、国际触达
- **频率**：有内容就发，最多一天一条 Thread
- **语言**：中英文都可，英文触达更广

### 小红书
- **吸收**：经验、感悟、对比、生活化话题
- **回报**：中文泛流量、女性 + 职场人群
- **频率**：1-3 篇/周
- **语言**：中文，口语化

### 抖音
- **吸收**：能口播 2-3 分钟的核心观点
- **回报**：大众触达、但技术内容门槛高
- **频率**：按录制能力定
- **语言**：中文

---

## 常见误判

- **"技术深度文也发小红书吧"** → 不要。小红书用户不看，数据会惨。除非你改写得非常降维。
- **"什么都发 X Thread"** → 不要。X Thread 有一定"门槛感"，水内容会掉粉。
- **"这个八卦我写 blog 吧"** → 不要。blog 留存长，写八卦对作者长期 IP 不利。
- **"抖音要做就每天发"** → 不要。技术内容抖音做不出量，别逼自己。

## 给 scout report 的建议字段

```yaml
platforms:
  - blog          # 默认 on，技术内容必选
  - x             # 如果有国际受众或硬观点
  - xiaohongshu   # 如果有中文泛受众 + 能降维
  - douyin        # 如果有清晰口播脚本素材
```

不要把"平台全开"当默认。**选得少，做得精**。
